Fast Entry A/P Invoice

Overview

The Versino Fast Entry Module is a specialized tool for the fast entry of A/P invoices in SAP Business One. It is ideal for companies that regularly receive similar invoices from the same vendors. Users can create new invoices with just a few clicks based on already existing purchasing documents (A/P invoices, goods receipts, purchase orders).

Module Access: You will find the fast entry in the menu under:

  • Purchasing > Fast Entry A/P Invoice
  • Versino Financial Suite > Fast Entry A/P Invoice

At the first use of the module, two user-defined fields are automatically created in the A/P invoices that are needed for the favorites management.

Main Features

Fast Document Entry & Amount Distribution

The core function is the creation of new invoice drafts by copying data from an existing document. If the new invoice amount differs from the original document, the system distributes the new amount proportionally across all rows.

Intelligent Search Functions

A full-text search helps to quickly find the right base document. You can switch between a document search and a row search:

Document Search Searches

  • Document comments
  • Favorite descriptions
  • External document numbers
  • Document numbers
  • Document IDs

Row Search Searches

  • Item codes
  • Item descriptions

Search syntax: Separate multiple terms with semicolon, e.g. OfficeSupplies;Paper;Toner — wildcards are used automatically, case is ignored.

Color Highlighting of Search Results

The results are highlighted by color according to relevance:

  • Green: Documents marked as favorites (always sorted at the top)
  • Cyan: High relevance (more than 50% of search terms found)
  • Yellow: Standard documents without search terms
  • White: Normal relevance
[IMAGE: Search result matrix with all four color codes (Green/Cyan/Yellow/White) visible at the same time]

Favorites Management

Frequently used documents can be marked as favorites directly in the A/P invoice. They always appear at the top of the search results (green), which significantly speeds up the selection of templates. At the first module start, two UDF fields are automatically created:

  • Document marked as favorite (Yes/No selection)
  • Favorite remark (text field for descriptions)
[IMAGE: A/P invoice with the two UDF fields "Document marked as favorite" and "Favorite remark"]

Multi-Database Support

The module can access multiple SAP Business One databases to use a document from one database as a template for a new invoice in another database.

  • Home database: Full functionality with links to original documents and automatic opening of created drafts.
  • Foreign databases: Document creation as draft, with notification messages instead of automatic opening. Link buttons are hidden.
[IMAGE: Multi-DB login dialog when switching to another database]

Calculation Example: Proportional Amount Distribution

Distribution with Differing Invoice Amount

Original document: 1,000 EUR with 3 rows (400 / 300 / 300 EUR gross)

New invoice: 1,200 EUR

Proportion factor: 1.2

New distribution: 480 / 360 / 360 EUR gross

Tax rates per row are taken into account, rounding differences are handled automatically.

[IMAGE: Schematic representation of the proportional amount distribution (1,000→1,200, factor 1.2)]

Payment Terms Calculation

The due date is calculated automatically based on the vendor's payment terms. The following modes are supported:

  • Normal: Invoice date + X days
  • Beginning of month: To the 1st of the month + extra days/months
  • Mid-month: To the 15th or end of month + extra days/months
  • End of month: To the end of month + extra days/months

Integration with Other VPS Modules

VPS Posting Text Configurator (JE Text Setter)

Uses automatically generated posting texts from the JE Text Setter so that the invoices created with Fast Entry receive uniform, meaningful texts.

VPS DATEV Export

The A/P invoices created with Fast Entry are treated by the DATEV Export like regular invoices and classified correctly.

Application

  1. Navigate to Purchasing > Fast Entry A/P Invoice.
  2. Choose the vendor. The payment terms and the date are filled automatically.
  3. Enter the vendor invoice number and the gross amount of the new invoice.
  4. Use the search field to look for a matching template (e.g. by item or project). Favorites are automatically prioritized.
  5. Select the matching document in the result list.
  6. Click on "Add". The system creates a new invoice draft with the transferred data and the new, proportionally distributed amount.

Tips and Troubleshooting

Tips for Daily Work

  • Use favorites: Mark recurring invoices (e.g. rent, leasing) as favorites to find them immediately.
  • Search efficiently: Use specific search terms and separate multiple terms with a semicolon (;). Switch between document and row search as needed.
  • Multi-DB efficiency: Use the central document search across multiple companies. Note: Home database offers full functionality, foreign databases create documents only as drafts.
  • Check amount distribution: Check the calculated due dates and the proportional distribution with differing sums before creating the document.

Common Problems and Solutions

Problem: A vendor is not found.

Solution: Check the spelling. The search works with both the vendor code and the name.

Problem: No search results displayed.

Solution: Broaden the search terms or switch the search type (document search vs. row search).

Problem: Favorites fields are not visible.

Solution: Start Fast Entry once — the UDF fields "Document marked as favorite" and "Favorite remark" are created automatically at the first start.

Problem: Multi-database login fails.

Solution: Check the username and password for the target database. Note that passwords are not stored for security reasons.

Problem: The due date is incorrect.

Solution: The date is calculated automatically based on the payment terms stored for the vendor. Check and correct them if necessary in the business partner master data.

Problem: The proportional distribution does not seem to be correct.

Solution: Check the entered gross amount. The distribution is based on the row shares of the original document. Any rounding difference is automatically posted to the first row.

Problem: Document creation produces an error.

Solution: Check the vendor invoice number and invoice amount. Make sure the posting period is open and that you have the necessary permissions.
