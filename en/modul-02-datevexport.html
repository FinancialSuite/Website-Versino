DATEV Export

Overview

The Versino DATEV Export Module is the central interface for exporting SAP Business One data to the DATEV format. It enables a complete data exchange — including postings, master data (customers, vendors, G/L accounts) and payment information — and is specifically optimized for German accounting requirements.

Module Access: The export dialog is found under Versino Financial Suite > DATEV Export. The export logs can be viewed under Versino Financial Suite > Logs > Export Log.

Benefits of the Module

  • Comprehensive Data Integration: Exports not only journal entries but also all relevant master data and payment terms for a complete data handover.
  • Smart Status Management: Effectively prevents duplicate exports through automatic status tracking for each posting and ensures complete traceability.
  • Quality Assurance: Detailed logging, validation prior to export and transparent error handling secure data quality.

Main Features

Comprehensive Data Export

The module exports not only journal entries but also all relevant master data, ensuring a complete handover to your tax advisor:

  • Journal entries
  • Customer and vendor master data
  • G/L account labels
  • Payment terms

Supported Posting Types

The module uses specialized exporters for the various posting types from SAP Business One:

  • Journal entries — Single postings with no special handling
  • A/R invoices (VKRE) and A/R credit memos (VKGS)
  • A/P invoices (EKRE) and A/P credit memos (EKGS)
  • Down-payment invoices for sales and purchasing
  • Incoming and outgoing payments
  • Goods receipts (EKWE) and deliveries (VKLS)
  • Returns for purchasing and sales
  • Inventory postings (stock count, transfers, revaluations)
  • Production orders, reconciliations, check submissions
  • Service postings and reversal postings (with special handling)

Export Status System

Each posting receives a status that is shown directly in the journal entry:

  • Y — Successfully exported
  • N — Not yet exported
  • E — Excluded from export
  • F — Export failed

The status can be changed manually at any time via a separate dialog, e.g. to enable a re-export.

Detailed Filtering Options

For posting export, extensive filters are available. For the posting status filter, you can choose between:

  • Only not exported
  • Not exported and failed
  • Only failed
  • All except failed
  • All postings

Logging and Monitoring

All export operations are logged in detail in a Log tab, with color-coded messages (success, warning, error) and a breakdown by module categories (general, customers/vendors, G/L accounts, payments, journal entries).

Export Files (DATEV Standard)

The export creates CSV files following the DATEV standard with semicolon as field separator:

Generated Files

  • EXTF_Buchungsstapel.csv — Main file with all posting rows (over 100 fields per row)
  • EXTF_Debitoren.csv — Customer master data
  • EXTF_Kreditoren.csv — Vendor master data
  • EXTF_Kontenbeschriftungen.csv — G/L accounts
  • EXTF_Zahlungsbedingungen.csv — Payment terms

All files contain the DATEV version header and can be imported directly into DATEV programs.

Integration with Other VPS Modules

VPS Posting Text Configurator (JE Text Setter)

The JE Text Setter generates uniform, meaningful posting texts. These significantly improve the readability of the DATEV files and reduce queries from your tax advisor. Texts are automatically adjusted to DATEV length limits and special characters are avoided.

VPS Pro-Rata

Pro-Rata accrual/deferral postings are handled specially by the DATEV export: automatic recognition as accrual/deferral postings, correct reference to the original document and optimized DATEV classification.

VPS Main Module (Configuration)

DATEV client number, consultant number, account length and chart of accounts are configured centrally in the main module and automatically used by the export.

Application: Step-by-Step

Exporting Postings and Master Data

  1. Navigate to Versino Financial Suite > DATEV Export.
  2. Select the data types to export (e.g. journal entries, customers/vendors).
  3. Define the desired period and the date type (posting date or document date).
  4. Set the filter for "Posting Status" to "Only not exported" to avoid duplicates.
  5. Choose the target directory for the export file.
  6. Click "Start Export". The postings are exported and automatically marked as "exported" (status Y) in SAP B1.

Reviewing the Export Log

After the export, switch to the Log tab. Here you see a detailed, color-coded record of all operations and can analyze any error messages.

Manually Changing the Export Status

Open the affected journal entry in SAP Business One. In the "Financial Suite" tab, you can manually adjust the export status — for example to enable a re-export or to deliberately exclude a posting from the export (status E).

Tips and Troubleshooting

Tips for Daily Work

  • Export strategy: Run regular exports (e.g. monthly) to keep the data volume manageable. Use meaningful export paths for better organization.
  • Quality check: After every export, review the log for warnings or errors. Validate exports with small periods before processing larger volumes.
  • Status management: Use the status filters specifically. To correct a faulty export, manually reset the status to N and re-export only the failed postings.
  • Activate JE Text Setter: Before the export, ideally activate the JE Text Setter for uniform DATEV-optimized posting texts.

Common Problems and Solutions

Problem: The export fails with a validation error.

Solution: Check the log for details. Common causes are closed posting periods, missing account or tax assignments, or inconsistent charts of accounts. Analyze the error message in the Log tab.

Problem: The created export files are incomplete or faulty.

Solution: Make sure you have write access to the chosen export directory and that there is enough disk space. Check the export period and the CSV structure.

Problem: The export status is not updated for a posting.

Solution: Check your user permissions for changing journal entries. Make sure the add-on is correctly installed. As a fallback, the status can be changed manually in the journal entry.

Problem: Certain posting types are not exported.

Solution: Check the posting-type-specific exporters. Make sure all required data (e.g. account assignment, tax rate) is present and verify the configuration for special posting types.

Problem: The export dialog does not open.

Solution: Check your user permissions for the module. Make sure the Financial Suite has loaded correctly and the menu item is active.
