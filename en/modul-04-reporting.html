Reporting Package

Overview

The Versino Reporting Module is the central reporting system of the Financial Suite. It offers a comprehensive collection of pre-built Crystal Reports and works fully in the background to provide you with the right analyses. A key advantage is that the module automatically detects whether you are using a Microsoft SQL Server or SAP HANA database and loads the corresponding performance-optimized reports.

The module starts asynchronously without blocking the SAP Business One UI and automatically detects your system language. Access to the reports: All reports can be found in the SAP Business One menu under Versino Financial Suite > Financial Suite Reports.

Main Features

The module offers two types of reports to cover different requirements: Standard reports for quick access and mass reports with upstream filter dialogs.

Standard Reports (direct start)

These reports can be started directly and provide quick analyses. Filtering (e.g. by date) is done directly in the Crystal Reports viewer.

  • Cash Book — Shows all cash movements chronologically with running balance.
  • G/L Account Statement — Detailed movement lists for selected G/L accounts.
  • Open Items Customers (Release 07 version) — Overview of unpaid receivables with due-date analysis.
  • Trial Balance (TB) — Compact account-oriented overview of all accounts.
  • Trial Balance BP — Business-partner-oriented variant.
  • Top 10 Customers — Ranking by sales volume.
  • Top 10 Vendors — Ranking by purchasing volume.
  • Payment Behavior — Analysis of payment behavior with average payment durations.
  • BWA — Monthly business analysis according to standard BWA scheme.

Mass Reports (with filters)

These reports offer special filter dialogs before start in order to perform targeted mass processing. The system creates a separate file for each selected dataset.

  • Balance Confirmation — Creates separate confirmation letters for any number of customers as of a selectable key date.
  • Netting Reconciliations — Documents performed netting operations for a date range.
  • Payment Advice — Creates detailed payment advices for a specific payment run (only status "Executed").

Filter Operation

  • Date filter: Click the calendar icon for date selection.
  • Choose-From-List: Magnifier icon opens the selection list (e.g. for business partners).
  • Dropdown: Automatically shows available values (e.g. payment runs).

Output Options

All reports support various output formats:

  • Preview — View in SAP Business One or in a separate window
  • Print — Direct printing
  • PDF — Export as PDF file
  • Excel — Export for further processing
  • Word — Export to Word format

For mass reports, individual file names are assigned automatically (e.g. Balance Confirmation [CustomerCode] - [CustomerName].pdf).

Database Adaptation (SQL Server vs. HANA)

The system automatically loads the right report version based on your database:

  • SQL Server: Uses optimized SQL queries.
  • HANA: Uses HANA-specific functions and in-memory processing for maximum performance.

Important Standard Reports in Detail

Top 10 Analyses

Identify your most important business partners at a glance — perfect for sales and purchasing analyses:

  • Top 10 Customers: Sorted by sales volume
  • Top 10 Vendors: Sorted by purchasing volume

Payment Behavior

Analyzes the payment behavior of your customers with average payment durations and trends. An ideal basis for dunning strategies and discount decisions.

Mass Reports in Detail

Balance Confirmation

Automatically creates balance confirmations for selected customers as of a key date. Filter options:

  • Key date: Date for the balance determination
  • Business partner: Selection via Choose-From-List

The system creates a separate file for each selected customer in the format Balance Confirmation [CustomerCode] - [CustomerName].pdf.

Netting Reconciliations

Documents performed netting operations between business partners. Filter options:

  • Reconciliation date from/to: Period of the netting operations
  • Business partner: Dropdown selection from netting partners

Payment Advice

Creates payment advices based on the SAP Business One payment wizard. The system automatically shows only payment runs with status "Executed", sorted by newest first.

Integration with Other VPS Modules

VPS Pro-Rata

Pro-Rata postings are automatically considered in all relevant reporting reports — in the cash book, the G/L account statement, the trial balance and in the balance sheet and P&L.

VPS Posting Text Configurator (JE Text Setter)

Standardized posting texts from the JE Text Setter automatically appear in all reports and significantly improve clarity — especially in the cash book and on G/L account statements.

VPS Main Module

The reporting module accesses central configuration data (e.g. tax settings, charts of accounts) and automatically adapts to the database version (SQL Server / HANA).

Application

Using a Standard Report (e.g. G/L Account Statement)

  1. Navigate to Versino Financial Suite > Financial Suite Reports > G/L Account Statement.
  2. A filter dialog opens. Enter the desired period, the G/L accounts and the display type (B1 form or as external Windows form).
  3. Click "OK". The report is generated.
  4. Use the export functions of the viewer to save the report.

Using a Mass Report (e.g. Balance Confirmation)

  1. Choose the report Balance Confirmation.
  2. Fill in the filter fields in the appearing dialog (key date, business partner via Choose-From-List).
  3. Start report creation. The system automatically creates separate files for each selected customer.

Tips and Troubleshooting

Tips for Daily Work

  • Report selection: Use standard reports for daily routine analyses, mass reports for targeted filtered analyses.
  • Performance: For very large data volumes, restrict filter criteria (e.g. shorter date range) as much as possible.
  • Multilingual: Report names and descriptions are automatically displayed in the SAP Business One UI language.
  • Export strategy: Export regularly to PDF/Excel for archiving and further processing.

Common Problems and Solutions

Problem: A report is empty or shows no data.

Solution: Check your filter settings, especially the date range. Make sure that there is actually data in the database for the chosen criteria.

Problem: Mass report does not create any files.

Solution: Check whether the filter criteria actually return records. For balance confirmations, for example, at least one business partner must be selected.

Problem: The reports start slowly.

Solution: The very first start of a report after opening SAP Business One can take a moment for initialization. This is normal — subsequent reports should load faster.

Problem: Crystal Reports does not work or there is an error message regarding the database connection.

Solution: Make sure the Crystal Reports components for SAP Business One are correctly installed. Check your database connection and restart SAP Business One. For persistent issues, contact your administrator.

Problem: BWA report does not load.

Solution: Make sure the financial report template BWA is created. You will find the menu under Financials > Financial Report Templates. If this is not the case, you can create it yourself (as shown in the image). The standard chart of accounts can be loaded automatically via button. For persistent issues, contact your administrator.
